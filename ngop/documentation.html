Enstore New Configuration Agent Help



Overview

This page shows the Enstore MIB ( Management Information Bases) information as a shortcut table. The MIB information is updated by the enstore new configuration agent. The shortcut table contains the links to the MIB information for each of the MIB groups.

The  requirements of the new configuration agent:

  • updating MIB by snmpwalk().
  • updating the node dictionaries by self-define functions.
  • updating the html files which can be posted on enstore web page.
  • checking the MIB such as tcp, udp, ip, icmp, snmp and ifeXtension group objects, sending the events and corresponding statistics to NGOP Central Server if any object value reaches the corresponding threshold.


The Functions of the MIB Groups:
 

  • The system group includes configuration information that pinpoints what the device is, where it is, and who to call when something goes wrong.
  • The tcp group lets a management application check up on some TCP configuration value such as the top limit on the number of concurrent tcp connections that a system can handle.
  • The udp group tracks traffic and errors.
  • The icmp group consists of counts  of every type of incoming and outgoing ICMP message.
  • The snmp group tracks everything and counts SNMP traffic and errors. especially, the snmp group tracks the number of incoming and outgoing gets, sets, responses, and traps, and counts various errors, such as attempts to read nonexistent variables.
  • The ip group covers the configuration and management needs of hosts and routers.
  • The ifeXtension group consists of several nodes. The management information includes configuration information, the status of the interface, incoming and outgoing traffic statistics, counts of  various kinds of errors, separate counts of broadcast and multicast traffic, variables that fire off a test and record the test results and the complete set of physical addresses for which an interface will absorb traffic.


To Develope the New Configuration Agent:

  • If you want to add a new node in ifeXtension group to monitor, please do: go to the separate file named node_init.py, add the node key,  node NAME,  node PRODucte state,  node CABLE number and node ORDER number in the node dictionary named node_d to initiate the node.
  • If you want to add a group of the MIB variables to monitor, please do: 1. go to the agent file named theAgent.py , 2. define a list that lists the variables,  3. call the functions get_mib_info() and setTable() to get the MIB information, 4. save the initiation of the group dictionary as the offset,  5.add the group name in the list named meNames, and  6.add the corresponding contents of checking  information in the checkNode() function.
  • To run this new configuration agent, put the new configuration agent file  theAgent.py and the file init_d.py in a subdirectory named config. Under the subdirectory config, do the following steps:
            1. setup ngop
            2. setenv PYTHONPATH $PYTHONPATH":$NGOP_DIR/ma"
                or
                export PYTHONPATH=$PYTHONPATH":$NGOP_DIR/ma"
            3. python theAgent.py &
