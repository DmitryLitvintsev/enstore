The D0En cluster is the Enstore system that D0 is using to access their Run II mass storage data. All nodes are Intel Lancewood (L440GX+) motherboards with dual P!!!s running Linux 2.2.13 kernels.

SUPPORT POLICY:

  • Fully supported 8 AM - 6 PM on weekdays.
  • Best effort support, (fix what can be fixed from home) 6 PM - 11 PM on weekdays, 8 AM- 6 PM on weekends and holidays.
  • No official support other times, special arrangements possible if notified in advance.
  • No official support during lab Christmas shutdown/layoff period.
  • All requests for support should go to helpdesk@fnal.gov or call the help desk at 2345 during normal hours or the operators during off hours. Alternatively, you can use the help desk web interface .
  • The help desk will contact the Integrated Systems and Development Department's Integrated Administration Group to resolve the problem. This group will be able to call upon enstore developers to help resolve unfamiliar problems.



The D0En cluster consists of the following nodes:

  • d0ensrv3 and d0ensrv5: Console Servers
  • d0ensrv1: Runs enstore file and volume clerk and pnfs namespace
  • d0ensrv2: Runs enstore configuration, logger, alarm and inquisitor servers.
  • d0ensrv4: Runs enstore library manager and media changer
  • d0enmvr4a-d0enmvr18a (aka d0enmvr4b-d0enmvr18b): Run enstore movers.
NO user areas on these machines are backed up, system files are backed up daily.



Enstore provides a generic interface for experimenters to efficiently use mass storage systems as easily as if they were native file systems.
